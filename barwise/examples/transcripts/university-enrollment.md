Facilitator: Walk me through how course enrollment works at the university.

Registrar: Students enroll in courses. Each student is identified by a student number that we assign when they're admitted.

Facilitator: Can a student enroll in multiple courses?

Registrar: Absolutely. A typical student takes four or five courses per semester. And each course has many students enrolled.

Facilitator: How are courses identified?

Registrar: Each course has a course code, like CS101 or HIST230. The code is unique across the catalog.

Facilitator: Does a course have other properties?

Registrar: A title and a credit value. The title is free text. Credits are always a whole number -- usually 3 or 4, but labs can be 1 or 2.

Facilitator: Who teaches the courses?

Registrar: Each course offering is taught by an instructor. An instructor is identified by their employee ID. Every course must have an assigned instructor before enrollment opens, but an instructor can teach multiple courses.

Facilitator: You said "course offering" -- is that different from a course?

Registrar: Good catch. A course is the catalog entry -- CS101, "Intro to Computer Science." A course offering is a specific instance of that course in a particular semester. So CS101 might have a Fall 2025 offering and a Spring 2026 offering. Students enroll in offerings, not courses directly.

Facilitator: How do you identify a semester?

Registrar: By term code. It's a short string like "F25" or "SP26." Each term has a start date and an end date.

Facilitator: Are there limits on enrollment?

Registrar: Yes. Each course offering has a maximum capacity. We never exceed it. And a student cannot enroll in the same offering twice.

Facilitator: What about grades?

Registrar: When a student completes an offering, they receive a letter grade. The possible values are A, B, C, D, and F. A grade is optional -- it's only assigned after the course ends.
